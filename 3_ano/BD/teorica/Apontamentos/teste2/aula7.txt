	View
. Relação virtual derivada de realção(ões) base.
. São vistas sobre dados de tabelas reais. (não existe segundo armazenamento de dados)
. Permite manipular os dados da view

Utiliza-se para:
	-> Apresentar dados (adaptar esquema da bd para aplicações)
	-> Questões de segurança de dados (integridade e privacidade)
	-> Ferramenta de estruturação de queries mais complexas

CREATE VIEW <view_name> AS <SQL_Query>

Pode ser usuada em:
	-> INSERT
	-> SELECT
	-> UPDATE
	-> DELETE

Aproximações (Depende da SGBD):
. Query modification -> transformação da query definida
. view materialization -> criação de uma tabela temporária com resultados da execução da view, sober a qual serão executadas as Operações SQL pretendidas


Alterar usa-se "ALTER"
Eliminar usa-se "DROP"

Em geral:
	é updatable se só tiver só uma tabela com primary key e atributos todos os atributos com NOT NULL e sem default value
	não é updatable se usar várias tabela (joins) ou utilizarem group bys ou funções de agregação.

Não se recomenda modificar dados com views

CHECK OPTION -> final da definição da view
	-> utilizada se quiser garantir que as condições de clásula WHERE são verificadas na atualização
