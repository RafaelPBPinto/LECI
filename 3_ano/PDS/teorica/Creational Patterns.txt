	Creational Patterns

Factory Method - construtor virtual
	interface para criar um objeto mas deixar à responsabilidade das subclasses decidirem qual classe instanciar


Abstract Factory - várias famílias de objetos enquanto Factory para uma epecífica e as suas subclasses 

Builder - Separa a contrução de objeto complexo da sua representação
	Dividir representação complexa

Singleton - Assegurar que classe só tem uma instância
	- usar método getInstance para buscar instância
		- quando chamado pela primeira vez cria instância

Object Pool - Deve ser criada para objetos pesados
	- acquire e release methods 
	- guardar objetos reutilizaveis

Prototype - Especificar tipo de objeto a ser criado usando uma instância protótipo, e criar novos objetos copiando o protótipo

	Registo que guarde protótipos (Mapa proto, new a ser usado)
	Criar factory que recebe qual prototipo criar
